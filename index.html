⚡ Zero-Copy Networking & Memory-Mapped I/O

Tech Insight: Eliminate data copies for maximum throughput! Traditional network stacks copy data multiple times (kernel ↔ user space), but zero-copy techniques like mmap(), sendfile, and DMA bypass these unnecessary transfers. Modern systems achieve 10x performance gains by letting hardware write directly to/from device buffers.

Memory-mapped files let you treat disk as RAM — the OS handles paging transparently. When combined with zero-copy network APIs (like Linux's sendfile or DPDK), data flows from storage to NIC without touching your application's memory space at all. This is how high-frequency trading systems and video streaming platforms achieve line-rate throughput.

# 🚫 SLOW: Traditional I/O with multiple copies
def send_file_traditional(src_path, dst_socket):
    with open(src_path, 'rb') as f:
        data = f.read(65536)  # Copy 1: disk → kernel buffer
    while data:               # Copy 2: kernel → user space
        packed_data = data.encode()  # Copy 3: format conversion
        dst_socket.send(packed_data)
        data = f.read(65536)
    # Total: 4-8 memory copies per packet!

# ✅ FAST: sendfile zero-copy (Linux)
def send_file_zerocopy(src_path, dst_fd):
    with open(src_path, 'rb') as src:
        os.sendfile(dst_fd, src, None)  # Direct disk→NIC transfer!
    # Total: 1 memory copy only! Hardware handles the rest.

# 🎯 Memory-mapped file access (disk looks like RAM):
import mmap
def process_large_file(filepath):
    with open(filepath, 'r+b') as f:
        mm = mmap.mmap(f.fileno(), 0)  # Map entire file to virtual memory!
        pattern_count = mm.count(b'ERROR')  # Direct buffer access
        print(f"Found {pattern_count} errors")
        del mm  # OS syncs dirty pages automatically

Pro Tip: For Rust developers, check out the mmap-rust, memfd, and DPDK crates. The kernel's page cache acts as a universal buffer pool — smart zero-copy design reuses already-loaded pages instead of creating new copies. This is why Elasticsearch can index terabytes using only hundreds of GBs of RAM: it never owns the data, just pointers to OS-managed memory.
