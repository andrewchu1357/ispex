🗂 Git's Object Database Secret

Did you know? Git doesn't store files like a normal system. Everything is hashed content-addressable objects in `.git/objects`. Your "files" are reconstructed on-the-fly from blobs, trees, and commits.

That's why `git fsck` can verify your entire repo integrity — it just checks object references!

# Find what changed between commits (it's all objects!):
git diff-tree --no-commit-id --name-only -r HEAD~1

# See raw object types:
git cat-file -t 
  blobs    → file contents
  trees    → directory listings  
  commits  → snapshots pointing to root tree

Pro Tip: Git LFS exists because storing large binary blobs directly bloats your object database. It replaces binaries with pointer text files, keeping the repo snappy.
